Manager Read Me: (Last update 01-22-2026)

GameManager.js handles all of the syncing of all players interacting with any of the puzzles, as well as handles syncing for late joiners. DO NOT MODIFY ANY FUNCTION UNLESS NECESSARY. However, adding new function calls within the event listeners and ‘socket.on’ functions shouldn’t affect the underlying logic of the GameManager.js. 

As long as the puzzles are set up properly and the AFrame entities have the correct ID naming conventions, all logic should work as intended. Please read the README found in each subfolder as it provides a more detailed explanation on how to set up each puzzle.

IMPORTANT: Ensure to have an AFrame entity that has the ‘gamermanager’ component for socket communication. 

In each document will be an update section that will state most recent updates (if any), before adding it within the original text of the READMEs.
